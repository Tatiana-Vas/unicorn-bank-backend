# Parent Home (My Kids)

## Назначение экрана
Главный экран родителя после логина. Экран служит **центром управления детьми и входящих запросов**.

Экран открывается:
- после welcome (для уже зарегистрированного пользователя)
- при переходе на вкладку **Home** (Bottom Navigation)

---

## Основные блоки экрана

### 1. Header
- Аватар родителя
- Приветствие ("Hi, Maksim")
- Иконка уведомлений (badge при наличии новых событий)

---

### 2. My Kids (карточки детей)
Горизонтальный список карточек детей + карточка добавления нового ребёнка.

**Состояния карточек:**
- Активная карточка — визуально выделена
- Неактивные карточки — обычное состояние

**Поведение:**
- **Один тап по карточке ребёнка** → переключение активного ребёнка **на этом же экране**
- Активный ребёнок становится контекстом для всех данных ниже (Requests, Events, Accounts и др.)
- **Переход в Kid Profile**:
  или через long tap по карточке

---

### 3. Requests
Блок входящих запросов от активного ребёнка.

**Типы запросов:**
- Cash withdrawal request
- New deposit request
- Spending request (на покупку)
- New task request
- Task approval request

**Поведение:**
- Тап по запросу → переход сразу на соответствующий экран действия:
  - выдать наличные
  - создать / пополнить депозит
  - выдать деньги на покупку
  - создать новый таск
  - апрув / отклонить выполненный таск

**UC:**
- **UC-PARENT-HOME-04: Process requests**

**Empty state:**
- Если запросов нет → отображать сообщение:
  > "Нет новых запросов"

---

### 4. Events
Список ближайших событий, связанных со всеми детьми.

Примеры:
- Закрытие депозита (с датой)
- День рождения ребёнка
- Запланированные финансовые события

**Поведение:**
- Информационный блок
- Тап по событию → переход к деталям (опционально)

---

## Bottom Navigation

### Вкладки:
1. **Home** — My Kids (текущий экран)
2. **Kids Profile** — профиль активного ребёнка
3. **Accounts** — счета активного ребёнка
4. **Requests** — список запросов ребенка
5. **Tasks** — задания активного ребёнка

> Контекст активного ребёнка сохраняется при переключении вкладок.

---

## Empty & Edge cases

- Нет детей:
  - отображается только карточка **Add child**
- Один ребёнок:
  - карточка всегда активна, переключения нет
- Несколько детей:
  - активный ребёнок явно подсвечен

---

## Связанные use cases
- UC-PARENT-HOME-01: View kids
- UC-PARENT-HOME-02: Switch active child
- UC-PARENT-HOME-03: Navigate to kid profile
- UC-PARENT-HOME-04: Process requests
- UC-PARENT-HOME-05: View upcoming events
